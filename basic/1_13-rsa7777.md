# 1.13: RSA7777

## Description

SSH service running on port 7777. A public key is provided. Get in.

## Solution

Tried connecting first just to see the response:

```bash
ssh -p 7777 user@target
# Permission denied (publickey)
```

Only public key auth, as expected. The challenge gives the corresponding public key, so the goal is to recover the private key from it.

Inspected the key:

```bash
openssl rsa -pubin -in id_rsa.pub -text -noout
# Public-Key: (1024 bit)
# Modulus: ...
# Exponent: 65537
```

1024-bit key is already on the weak side. Ran it through `RsaCtfTool`:

```bash
python3 RsaCtfTool.py --publickey id_rsa.pub --private
```

It tried Fermat factorization and succeeded. Fermat's method works when p and q are close to each other, meaning their difference is small relative to their size. With a 1024-bit key that was clearly generated carelessly, it factored in seconds.

Saved the output private key:

```bash
python3 RsaCtfTool.py --publickey id_rsa.pub --private > id_rsa_recovered
chmod 600 id_rsa_recovered
ssh -i id_rsa_recovered -p 7777 user@target
```

Got a shell, found the flag in the home directory.

Flag: `33a4a797ac2146252c32c18dfa850a14`
